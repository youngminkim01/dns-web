### 1️⃣ 서버 설정 확인
`server-config.js` 파일에서 SSH 연결 정보를 확인하세요:

```javascript

    host: '192.168.10.247',      // DNS 서버 IP

    username: 'apro',
    password: 'aproit1!',
    tryKeyboard: true

```

```bash
cd DNS-Management-System
npm start
```

또는 개발 모드 (자동 재시작):
```bash
npm run dev
```

### 3️⃣ 웹 브라우저에서 접속
```
http://localhost:3000
```

## ✨ 새로운 기능

### 🔄 실시간 서버 연동
- 페이지 로드 시 실제 DNS 서버에서 레코드 자동 조회
- 레코드 추가/수정/삭제 시 즉시 서버에 반영
- BIND9 자동 재로드

### 📊 지원하는 레코드 타입
- **A**: IPv4 주소
- **AAAA**: IPv6 주소
- **CNAME**: 별칭
- **MX**: 메일 서버 (우선순위 포함)
- **TXT**: 텍스트 레코드
- **NS**: 네임서버
- **PTR**: 역방향 조회

## 🛠️ 문제 해결

### SSH 연결 실패
```
✗ SSH 연결 실패: Connection refused
```

**해결 방법**:
1. DNS 서버가 실행 중인지 확인
2. 방화벽에서 22번 포트 허용 확인
3. SSH 서비스 실행 확인: `sudo systemctl status ssh`
4. `server-config.js`의 연결 정보 확인

### DNS 레코드 조회 실패
```
DNS 레코드 조회 중 오류가 발생했습니다.
```

**해결 방법**:
1. 존 파일 경로 확인: `/etc/bind/zones/db.aproele.com`
2. 파일 권한 확인: `ls -l /etc/bind/zones/`
3. BIND9 서비스 상태 확인: `sudo systemctl status bind9`

### BIND9 재로드 실패
```
✗ BIND9 재로드 오류: Command failed
```

**해결 방법**:
1. BIND9 설정 파일 검증: `sudo named-checkconf`
2. 존 파일 검증: `sudo named-checkzone aproele.com /etc/bind/zones/db.aproele.com`
3. BIND9 로그 확인: `sudo tail -f /var/log/syslog | grep named`

## 🎓 사용 예시

### DNS 레코드 추가
1. 대시보드에서 "새 레코드 추가" 클릭
2. 레코드 정보 입력:
   - 타입: A
   - 이름: www.aproele.com
   - 값: 192.168.1.100
   - TTL: 3600
3. "저장" 클릭
4. ✅ "DNS 레코드가 성공적으로 추가되었습니다." 메시지 확인

### DNS 레코드 수정
1. 레코드 목록에서 수정할 레코드의 ✏️ 버튼 클릭
2. 값 변경 (예: 192.168.1.200)
3. "저장" 클릭
4. ✅ "DNS 레코드가 성공적으로 수정되었습니다." 메시지 확인

### DNS 레코드 삭제
1. 레코드 목록에서 삭제할 레코드의 🗑️ 버튼 클릭
2. 확인 대화상자에서 "확인" 클릭
3. ✅ "DNS 레코드가 성공적으로 삭제되었습니다." 메시지 확인
