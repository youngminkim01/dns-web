# 🚀 빠른 시작 가이드

DNS 관리 시스템을 5분 안에 시작하세요!

---

## ⚡ 3단계로 시작하기

### 1️⃣ 의존성 설치 (최초 1회만)

PowerShell 또는 명령 프롬프트를 열고:

```bash
cd c:\Users\apro\DNS\DNS-Management-System
npm install
```

**PowerShell 실행 정책 오류가 발생하면:**
```powershell
powershell -ExecutionPolicy Bypass -Command "npm install"
```

---

### 2️⃣ 서버 실행

```bash
npm start
```

**또는:**
```powershell
powershell -ExecutionPolicy Bypass -Command "npm start"
```

**성공 메시지:**
```
DNS 관리 서버가 포트 3000에서 실행 중입니다.
http://localhost:3000 에서 접속하세요.

기본 로그인 정보:
사용자명: admin
비밀번호: admin123
```

---

### 3️⃣ 브라우저로 접속

1. 웹 브라우저 열기 (Chrome, Firefox, Edge 등)
2. 주소창에 입력: **`http://localhost:3000`**
3. 로그인:
   - **사용자명**: `admin`
   - **비밀번호**: `admin123`

---

## 📝 첫 DNS 레코드 추가하기

### 단계별 가이드:

1. **로그인 후 대시보드에서 "새 레코드 추가" 버튼 클릭**

2. **레코드 정보 입력:**
   ```
   레코드 타입: A
   레코드 이름: www.example.com
   값: 192.168.1.100
   TTL: 3600
   ```

3. **"저장" 버튼 클릭**

4. **완료!** 레코드 목록에서 확인 가능

## 🆘 문제 해결

### ❌ "npm을 찾을 수 없습니다"

**해결방법:**
1. Node.js 설치: https://nodejs.org
2. 설치 후 터미널 재시작
3. `node --version` 명령으로 확인

---

### ❌ "포트 3000이 이미 사용 중입니다"

**해결방법 1: 다른 프로그램 종료**
```bash
# Windows
netstat -ano | findstr :3000
# 표시된 PID 확인 후 작업 관리자에서 종료
```

**해결방법 2: 포트 변경**
`server.js` 파일에서:
```javascript
const PORT = 3000; // 3001, 8080 등으로 변경
```

---

### ❌ 로그인이 안 됩니다

**확인사항:**
- 사용자명: `admin` (소문자)
- 비밀번호: `admin123` (숫자 포함)
- 브라우저 캐시 삭제 후 재시도

## 🎓 레코드 타입 치트시트

### A 레코드 (가장 많이 사용)
```
이름: www.example.com
값: 192.168.1.100
용도: 도메인을 IPv4 주소로 연결
```

### CNAME 레코드
```
이름: blog.example.com
값: www.example.com
용도: 도메인 별칭 (다른 도메인으로 리다이렉트)
```

### MX 레코드
```
이름: example.com
값: mail.example.com
우선순위: 10
용도: 메일 서버 지정
```

### TXT 레코드
```
이름: example.com
값: v=spf1 include:_spf.example.com ~all
용도: SPF, DKIM, 도메인 인증 등
```

## 💡 팁

### 🔥 자주 사용하는 TTL 값
- **300** (5분): 자주 변경되는 레코드
- **3600** (1시간): 일반적인 레코드 (기본값)
- **86400** (24시간): 거의 변경되지 않는 레코드

### 🎯 레코드 이름 작성 팁
- **루트 도메인**: `example.com` 또는 `@`
- **서브도메인**: `www.example.com`, `mail.example.com`
- **와일드카드**: `*.example.com` (모든 서브도메인)
