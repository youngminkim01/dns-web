### 1. 서버 측 개발 (server.js)

#### 1.1 SSH 연결 기능 추가
- **파일**: `DNS-Management-System/server.js`
- **변경 사항**:
  - `node-ssh` 라이브러리를 사용한 SSH 연결 구현
  - `server-config.js`에서 SSH 연결 정보 로드
  - SSH 연결 상태 관리 및 자동 재연결 기능
  - SSH 명령 실행 헬퍼 함수 구현

```javascript
// SSH 연결 함수
async function connectSSH() {
    if (sshConnected) return true;
    
    try {
        await ssh.connect(config.ssh);
        sshConnected = true;
        console.log('✓ SSH 연결 성공:', config.ssh.host);
        return true;
    } catch (error) {
        console.error('✗ SSH 연결 실패:', error.message);
        sshConnected = false;
        return false;
    }
}
```

#### 1.2 DNS 레코드 관리 함수 SSH 기반으로 변경

##### addRecordToZoneFile() - 레코드 추가
- 기존: 로컬 파일 시스템 사용
- 변경: SSH를 통해 원격 서버의 존 파일 읽기/쓰기
- 임시 파일 생성 후 sudo 권한으로 이동
- 파일 소유권 자동 설정 (bind:bind)

##### deleteRecordFromZoneFile() - 레코드 삭제
- SSH를 통해 존 파일 읽기
- 레코드 필터링 후 임시 파일로 저장
- sudo 권한으로 원본 파일 교체

##### incrementSerial() - 시리얼 번호 증가
- SSH로 존 파일 읽기
- 정규식으로 시리얼 번호 찾아 증가
- 변경된 내용을 SSH로 다시 저장

##### reloadBind() - BIND9 재로드
- 기존: 로컬 exec() 사용
- 변경: SSH를 통한 원격 `sudo rndc reload` 실행
- 에러 처리 개선

##### parseZoneFile() - 존 파일 파싱
- SSH를 통해 존 파일 읽기
- SOA, NS 레코드 제외 (관리 레코드)
- 각 레코드에 고유 ID 부여
- MX 레코드의 우선순위 추출
- status 필드 자동 추가 ('active')

### 2. 클라이언트 측 개발 (dashboard.js)

#### 2.1 서버에서 DNS 레코드 불러오기 기능 추가
- **함수**: `loadDNSRecords()`
- **기능**:
  - `/api/dns/records?zone=aproele.com` 엔드포인트 호출
  - 서버에서 실제 DNS 레코드 조회
  - 연결 실패 시 샘플 데이터로 폴백
  - 콘솔에 상태 메시지 출력

```javascript
async function loadDNSRecords() {
    try {
        const response = await fetch('/api/dns/records?zone=aproele.com');
        
        if (!response.ok) {
            throw new Error('DNS 레코드 조회 실패');
        }
        
        const data = await response.json();
        
        if (data.success && data.records) {
            dnsRecords = data.records;
            renderRecords();
            console.log('✓ DNS 레코드를 서버에서 불러왔습니다:', dnsRecords.length, '개');
        }
    } catch (error) {
        console.error('DNS 레코드 불러오기 오류:', error);
        // 오류 발생 시 샘플 데이터 사용
        dnsRecords = [샘플 데이터...];
        renderRecords();
        console.warn('⚠ 서버 연결 실패. 샘플 데이터를 사용합니다.');
    }
}
```

## 📊 API 엔드포인트

### 1. DNS 레코드 조회
- **URL**: `GET /api/dns/records?zone={zone_name}`
- **응답**:
```json
{
  "success": true,
  "records": [
    {
      "id": 1,
      "type": "A",
      "name": "www",
      "value": "192.168.1.100",
      "ttl": 3600,
      "status": "active"
    }
  ]
}
```

### 2. DNS 레코드 추가
- **URL**: `POST /api/dns/add`
- **요청 본문**:
```json
{
  "type": "A",
  "name": "www.example.com",
  "value": "192.168.1.100",
  "ttl": 3600
}
```

### 3. DNS 레코드 수정
- **URL**: `PUT /api/dns/update/:id`
- **요청 본문**: (레코드 추가와 동일)

### 4. DNS 레코드 삭제
- **URL**: `DELETE /api/dns/delete`
- **요청 본문**:
```json
{
  "name": "www.example.com",
  "type": "A",
  "value": "192.168.1.100"
}
```

## 🔐 보안 고려사항

### SSH 연결
- 비밀번호 인증 사용 (현재)
- 향후 SSH 키 기반 인증으로 전환 권장
- 연결 정보는 `server-config.js`에 분리 저장

### 파일 권한
- 모든 BIND9 파일 작업에 `sudo` 사용
- 파일 소유권 자동 설정 (`bind:bind`)
- 임시 파일은 `/tmp` 디렉토리 사용

### 입력 검증
- 향후 레코드 값 검증 로직 추가 필요
- SQL Injection 방지 (현재는 파일 기반이므로 해당 없음)
- XSS 방지를 위한 입력 이스케이프 처리

## 📞 문의 및 지원

문제가 발생하거나 질문이 있으시면 다음을 확인하세요:

1. **SSH 연결 문제**
   - `server-config.js`의 연결 정보 확인
   - 방화벽 설정 확인
   - SSH 서비스 실행 상태 확인

2. **BIND9 관련 문제**
   - `sudo systemctl status bind9` 로 서비스 상태 확인
   - `/var/log/syslog`에서 BIND9 로그 확인
   - `sudo named-checkconf`로 설정 파일 검증

3. **웹 인터페이스 문제**
   - 브라우저 콘솔에서 에러 메시지 확인
   - 네트워크 탭에서 API 응답 확인
