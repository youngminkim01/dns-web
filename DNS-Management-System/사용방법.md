## 프로젝트 개요

### 🎯 목적
리눅스 BIND9 DNS 서버를 웹 브라우저에서 쉽게 관리할 수 있는 시스템입니다.

### ✨ 주요 기능
- ✅ 관리자 로그인 시스템
- ✅ DNS 레코드 추가/수정/삭제 (A, AAAA, CNAME, MX, TXT, NS, PTR)
- ✅ 실시간 레코드 검색
- ✅ DNS 통계 대시보드
- ✅ 모던하고 직관적인 UI

### 📁 파일 구조
```

├── index.html          # 로그인 페이지
├── dashboard.html      # 관리 대시보드
├── style.css           # 스타일시트
```

## 사용 방법

### 🔐 1. 로그인

1. 브라우저에서 `http://localhost:3000` 접속
2. 로그인 정보 입력:
   - **사용자명**: `admin`
   - **비밀번호**: `admin123`
3. "로그인 상태 유지" 체크박스 선택 (선택사항)
4. **로그인** 버튼 클릭

### 📝 2. DNS 레코드 추가

#### 단계별 가이드:

1. **대시보드 접속 후 "새 레코드 추가" 버튼 클릭**

2. **레코드 정보 입력**

3. **저장 버튼 클릭**

#### 레코드 타입별 예시:

**A 레코드 (IPv4 주소 매핑)**
```
레코드 타입: A
레코드 이름: www.example.com
값: 192.168.1.100
TTL: 3600
```

**AAAA 레코드 (IPv6 주소 매핑)**
```
레코드 타입: AAAA
레코드 이름: www.example.com
값: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
TTL: 3600
```

**CNAME 레코드 (별칭)**
```
레코드 타입: CNAME
레코드 이름: blog.example.com
값: www.example.com
TTL: 7200
```

**MX 레코드 (메일 서버)**
```
레코드 타입: MX
레코드 이름: example.com
값: mail.example.com
우선순위: 10
TTL: 3600
```

**TXT 레코드 (텍스트 정보)**
```
레코드 타입: TXT
레코드 이름: example.com
값: v=spf1 include:_spf.example.com ~all
TTL: 3600
```

**NS 레코드 (네임서버)**
```
레코드 타입: NS
레코드 이름: example.com
값: ns1.example.com
TTL: 86400
```

**PTR 레코드 (역방향 조회)**
```
레코드 타입: PTR
레코드 이름: 100.1.168.192.in-addr.arpa
값: www.example.com
TTL: 3600
```

### ✏️ 3. DNS 레코드 수정

1. 레코드 목록에서 수정할 레코드 찾기
2. 해당 레코드의 **연필 아이콘(✏️)** 클릭
3. 수정할 정보 변경
4. **저장** 버튼 클릭

### 🗑️ 4. DNS 레코드 삭제

1. 레코드 목록에서 삭제할 레코드 찾기
2. 해당 레코드의 **휴지통 아이콘(🗑️)** 클릭
3. 확인 대화상자에서 **확인** 클릭

### 🔍 5. 레코드 검색

상단 검색창에 검색어 입력:
- 도메인 이름으로 검색: `www.example.com`
- IP 주소로 검색: `192.168.1.100`
- 레코드 타입으로 검색: `A`, `CNAME`, `MX` 등

### 📊 6. 통계 확인

좌측 메뉴에서 **"통계"** 탭 클릭:
- **총 레코드**: 전체 DNS 레코드 수
- **DNS 존**: 관리 중인 DNS 존 수
- **활성 레코드**: 현재 활성화된 레코드 수

### 🚪 7. 로그아웃

우측 하단의 **로그아웃 버튼** 클릭

## 리눅스 DNS 서버 연동

### 🐧 BIND9 설치 (Ubuntu/Debian)

```bash
# 패키지 업데이트
sudo apt-get update

# BIND9 설치
sudo apt-get install bind9 bind9utils bind9-doc -y

# BIND9 상태 확인
sudo systemctl status bind9
```

### 📂 존 파일 디렉토리 생성

```bash
# 존 파일 디렉토리 생성
sudo mkdir -p /etc/bind/zones

# 권한 설정
sudo chown -R bind:bind /etc/bind/zones/
sudo chmod 775 /etc/bind/zones/
```

### 📄 예시 존 파일 생성

**파일 경로**: `/etc/bind/zones/db.example.com`

```bash
sudo nano /etc/bind/zones/db.example.com
```

**파일 내용**:
```bind
$TTL    604800
@       IN      SOA     ns1.example.com. admin.example.com. (
                              2024021001         ; Serial
                              604800             ; Refresh
                              86400              ; Retry
                              2419200            ; Expire
                              604800 )           ; Negative Cache TTL
;
; 네임서버 레코드
@       IN      NS      ns1.example.com.

; A 레코드
@       IN      A       192.168.1.1
ns1     IN      A       192.168.1.1
www     IN      A       192.168.1.100
mail    IN      A       192.168.1.101

; MX 레코드
@       IN      MX      10 mail.example.com.

; CNAME 레코드
blog    IN      CNAME   www.example.com.
```

### ⚙️ BIND9 설정

**파일 경로**: `/etc/bind/named.conf.local`

```bash
sudo nano /etc/bind/named.conf.local
```

**존 추가**:
```bind
zone "example.com" {
    type master;
    file "/etc/bind/zones/db.example.com";
};
```

### 🔧 권한 설정

```bash
# 사용자를 bind 그룹에 추가
sudo usermod -a -G bind $USER

# sudoers 설정 (rndc 명령 권한)
sudo visudo
```

**다음 줄 추가** (yourusername을 실제 사용자명으로 변경):
```
yourusername ALL=(ALL) NOPASSWD: /usr/sbin/rndc
```

### 🔄 BIND9 재시작

```bash
# 설정 파일 검증
sudo named-checkconf

# 존 파일 검증
sudo named-checkzone example.com /etc/bind/zones/db.example.com

# BIND9 재시작
sudo systemctl restart bind9

# 상태 확인
sudo systemctl status bind9
```

### 🧪 DNS 테스트

```bash
# 로컬 DNS 쿼리 테스트
nslookup www.example.com localhost

# dig 명령으로 테스트
dig @localhost www.example.com

# 특정 레코드 타입 조회
dig @localhost example.com MX
```

## 문제 해결

### ❌ 서버가 시작되지 않을 때

**1. Node.js 설치 확인**
```bash
node --version
npm --version
```

**2. 의존성 재설치**
```bash
rm -rf node_modules
npm install
```

**3. 포트 충돌 확인**
```bash
# Windows
netstat -ano | findstr :3000

# Linux
sudo lsof -i :3000
```

포트 3000이 사용 중이면 다른 프로세스 종료 또는 `server.js`에서 포트 변경

### ❌ 로그인이 안 될 때

**기본 계정 정보 재확인**:
- 사용자명: `admin`
- 비밀번호: `admin123`

**브라우저 캐시 삭제**:
- Chrome: `Ctrl + Shift + Delete`
- Firefox: `Ctrl + Shift + Delete`

**개발자 도구 확인**:
- `F12` 눌러서 콘솔 오류 확인

### ❌ DNS 레코드가 추가되지 않을 때

**1. 브라우저 콘솔 확인** (`F12`)

**2. 서버 로그 확인**
- 터미널에서 오류 메시지 확인

**3. BIND9 권한 확인**
```bash
ls -la /etc/bind/zones/
sudo chown -R bind:bind /etc/bind/zones/
```

**4. BIND9 로그 확인**
```bash
sudo tail -f /var/log/syslog | grep named
```

### ❌ BIND9 재로드 실패

**설정 파일 검증**:
```bash
sudo named-checkconf
sudo named-checkzone example.com /etc/bind/zones/db.example.com
```

**BIND9 재시작**:
```bash
sudo systemctl restart bind9
sudo systemctl status bind9
```

**로그 확인**:
```bash
sudo journalctl -u bind9 -n 50
```
