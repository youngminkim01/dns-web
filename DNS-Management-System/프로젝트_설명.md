## 🎯 프로젝트 목표

리눅스 DNS 서버(BIND9)를 구축한 후, 명령줄이 아닌 **웹 브라우저**에서 간편하게 DNS 레코드를 추가, 수정, 삭제할 수 있는 관리 시스템을 제공합니다.

---

## ✨ 주요 기능

### 1. 로그인 시스템
- 관리자 인증 기능
- 세션 관리 (로그인 상태 유지)
- 보안 로그아웃

### 2. DNS 레코드 관리
지원하는 레코드 타입:
- **A**: IPv4 주소 매핑
- **AAAA**: IPv6 주소 매핑
- **CNAME**: 도메인 별칭
- **MX**: 메일 서버 설정
- **TXT**: 텍스트 정보 (SPF, DKIM 등)
- **NS**: 네임서버 지정
- **PTR**: 역방향 DNS 조회

### 3. 관리 기능
- ✅ 레코드 추가
- ✅ 레코드 수정
- ✅ 레코드 삭제
- ✅ 실시간 검색
- ✅ 통계 대시보드

### 4. 사용자 인터페이스
- 🎨 모던한 다크 테마
- 📱 반응형 디자인 (모바일, 태블릿, PC 지원)
- ⚡ 부드러운 애니메이션
- 🔍 직관적인 검색 기능

## 🔄 작동 원리

### 전체 흐름도

```
사용자 브라우저
    ↓
[로그인 페이지] (index.html)
    ↓ (인증 성공)
[대시보드] (dashboard.html)
    ↓ (레코드 추가/수정/삭제)
[Express API] (server.js)
    ↓
[BIND9 존 파일 수정]
    ↓
[rndc reload]
    ↓
DNS 서버 업데이트 완료
```

### 상세 작동 과정

1. **로그인**
   - 사용자가 `index.html`에서 로그인
   - `login.js`가 인증 처리
   - 성공 시 세션 저장 후 `dashboard.html`로 이동

2. **레코드 추가**
   - 사용자가 "새 레코드 추가" 버튼 클릭
   - 모달 창에서 레코드 정보 입력
   - `dashboard.js`가 `/api/dns/add` API 호출
   - `server.js`가 BIND9 존 파일에 레코드 추가
   - 시리얼 번호 증가
   - `rndc reload`로 BIND9 재로드
   - 클라이언트에 성공 응답

3. **레코드 수정**
   - 사용자가 연필 아이콘 클릭
   - 기존 데이터가 모달에 표시
   - 수정 후 저장
   - `/api/dns/update/:id` API 호출
   - BIND9 존 파일 업데이트
   - 재로드

4. **레코드 삭제**
   - 사용자가 휴지통 아이콘 클릭
   - 확인 대화상자
   - `/api/dns/delete` API 호출
   - 존 파일에서 레코드 제거
   - 재로드

## 📊 API 엔드포인트

### 1. DNS 레코드 추가
```http
POST /api/dns/add
Content-Type: application/json

{
  "type": "A",
  "name": "www.example.com",
  "value": "192.168.1.100",
  "ttl": 3600
}
```

**응답**:
```json
{
  "success": true,
  "message": "DNS 레코드가 성공적으로 추가되었습니다.",
  "record": { ... }
}
```

### 2. DNS 레코드 수정
```http
PUT /api/dns/update/:id
Content-Type: application/json

{
  "type": "A",
  "name": "www.example.com",
  "value": "192.168.1.101",
  "ttl": 3600
}
```

### 3. DNS 레코드 삭제
```http
DELETE /api/dns/delete
Content-Type: application/json

{
  "name": "www.example.com",
  "type": "A",
  "value": "192.168.1.100"
}
```

### 4. DNS 레코드 조회
```http
GET /api/dns/records?zone=example.com
```

**응답**:
```json
{
  "success": true,
  "records": [
    {
      "name": "www",
      "ttl": "3600",
      "type": "A",
      "value": "192.168.1.100"
    }
  ]
}
```
