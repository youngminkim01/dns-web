# DNS 관리 웹 프로젝트 작업 총정리 (2026-02-10 ~ 2026-03-10)

본 문서는 사내 DNS(BIND) 관리를 위한 웹 대시보드를 구축하고 유지보수하며 진행했던 모든 주요 작업 내역을 요약한 문서입니다.

---

## 1. 프로젝트 개요
- **목적**: 리눅스 DNS 서버(BIND)의 환경설정 파일(존 파일)에 직접 접근하지 않고도, 웹 UI를 통해 레코드를 손쉽게 조회, 추가, 수정, 삭제할 수 있는 관리자 시스템 구축.
- **기술 스택**: Node.js, Express, HTML, CSS, Vanilla JS.
- **아키텍처**: Node.js 백엔드 서버가 대상 리눅스 서버에 SSH(또는 로컬 모드)로 접근하여 BIND `/etc/bind/zones` 하위의 존 파일을 파싱하고 조작합니다.

---

## 2. 주요 작업 연혁

### 🎯 2026-02-10: 초기 시스템 구축 및 연동 오류 해결
* **웹 서버 및 초기 UI 구축**: 로그인(`index.html`), 대시보드(`dashboard.html`), 서버(`server.js`) 토대 완성.
* **DNS 연동 버그 픽스**:
  * 프론트엔드 API 호출 시 파라미터 오류(`zones` -> `zone`) 수정
  * SSH을 이용한 원격 서버 파일 제어 시 `sudo` 권한 에러 해결 (비대화식 비밀번호 전달 방식 적용)
  * Windows 운영체제에서 서버 실행 시 Linux 원격 디렉터리 경로 기호(`\`)가 깨지는 현상 수정 (`path.posix.join` 적용)
  * 특정 BIND 레코드에서 TTL 값이 생략되었을 때 데이터 파싱이 어긋나는 버그 보완

### 🎯 2026-03-05: 사용자 접근성 개선 (포트 제거)
* **포트 번호 은닉화**: 사용자가 `dns.aproele.com:3000`과 같이 포트(3000)를 명시해야 하는 불편함을 해소하기 위하여, Nginx 프록시 구성 등의 변경을 통해 도메인만으로 접속이 가능하게끔 구성(`nginx-dns-web.conf` 설정 참조 등).

### 🎯 2026-03-06: 대시보드 화면(UI) 최적화
* **불필요한 메뉴 제거**: 초기 버전에 있던 "통계(Statistics)" 영역 및 "DNS 존(Zones)" 관리 메뉴를 삭제.
* **사용성 집중**: "조회된 DNS 레코드" 메뉴 하나에만 집중하여, 심플하고 직관적으로 관리할 수 있도록 화면 개편.

### 🎯 2026-03-10: 폼 스타일링 오류 수정 및 최종 배포, 매뉴얼화
* **모달 폼 테마 안정화**: '새 레코드 추가' 시 나타나는 팝업 창에서 "레코드 타입(A)" 입력창에만 하얗게 들어가 있던 인라인 CSS 스타일을 제거하여, 다른 입력창(다크 테마)과 일관성을 갖도록 수정 (`dashboard.html`).
* **운영 서버 배포 반영**: 수정한 HTML 및 JS 파일을 `deploy.js` 및 `deploy-start.js` 스크립트를 사용하여 원격 운영 서버(`192.168.10.243`)로 업로드하고 백그라운드 데몬 및 프로세스를 성공적으로 재시작.
* **관리자 매뉴얼 수립**: 운영자가 바로 시스템을 사용할 수 있도록 `docs/DNS_관리자_매뉴얼.md` 신규 작성.

## 3. 핵심 파일 및 디렉터리 구조 요약
* **`server.js`**: BIND 서버와 접속하여 존 파일을 조작하는 메인 백엔드 로직
* **`server-config.js`**: BIND 디렉터리 경로 및 SSH, 포트 등 구동 환경설정 변수 관리
* **`dashboard.html / dashboard.js / style.css`**: 클라이언트가 마주하는 핵심 웹 프론트엔드
* **`deploy.js / deploy-start.js`**: 작성된 소스코드를 운영계 DNS 서버로 릴리즈하고 재구동시키는 배포용 스크립트
* **`docs/` 폴더**: 관리자 매뉴얼 및 작업 히스토리(본 문서 등) 보관 장소
