# DNS 관리 웹 앱 — 서버 배포 방법

앱을 **리눅스 DNS 서버(192.168.10.243)** 에서 실행하면, 레코드 추가·삭제가 **해당 서버의 BIND에 바로 반영**됩니다.

---

## 1. 서버에서 실행할 때 (로컬 모드)

DNS 서버 **같은 머신**에서 Node 앱을 실행하면 SSH 없이 **로컬 파일**을 읽고 씁니다.

### 1) 파일 복사

- Windows의 `c:\Users\apro\DNS` 폴더 전체를 서버로 복사합니다.  
  (예: `scp -r c:\Users\apro\DNS apro@192.168.10.243:~/dns-web`)

### 2) 서버에서 설정 변경

서버에 올린 폴더 안에서:

```bash
cd ~/dns-web
cp server-config.server.example.js server-config.js
```

`server-config.js` 를 열어서 다음만 확인/수정합니다.

- **`localMode: true`** 로 두기 (이미 예시에 true 로 되어 있음)
- **`ssh.password`** : 서버에서 `sudo` 할 때 쓸 비밀번호 (예: apro 비밀번호)
- **`web.host: '0.0.0.0'`** : 다른 PC에서 접속하려면 `'0.0.0.0'` 유지

### 3) 실행

```bash
cd ~/dns-web
npm install
npm start
```

- 브라우저: **http://192.168.10.243:3000** (같은 네트워크의 PC에서 접속 가능)

이렇게 하면 **추가·삭제가 모두 이 서버의 BIND(/etc/bind/zones)** 에 적용됩니다.

---

## 2. Windows에서 실행할 때 (현재 방식)

- `server-config.js` 에서 **`localMode: false`** (기본값) 로 두고
- Windows에서 `npm start` 로 실행하면, 기존처럼 **SSH로 192.168.10.243** 에 접속해 존 파일을 수정합니다.

---

## 요약

| 실행 위치   | localMode | 동작 |
|------------|-----------|------|
| Windows PC | false     | SSH로 192.168.10.243 접속 → 존 파일 수정 |
| DNS 서버    | true      | 로컬에서 직접 /etc/bind/zones 읽기·쓰기 → 서버에 바로 반영 |

서버에도 적용되게 하려면 **서버에 앱을 올리고, 서버에서 `localMode: true` 로 실행**하면 됩니다.
