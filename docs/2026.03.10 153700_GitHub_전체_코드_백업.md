## 작업 내용
1. **로컬 환경 검토 및 설정**
   - 사용자 레포지토리 주소 확인: `https://github.com/youngminkim01/dns-web.git`
   - Git 실행 경로 (`C:\Program Files\Git\cmd\git.exe`)를 통해 윈도우 환경에서 로컬 저장소 초기화(`git init`) 진행.
   - `node_modules` 등 불필요하게 무거운 파일이 백업에 포함되지 않도록 `.gitignore` 파일을 작성하여 필터링 구성.

2. **초기 커밋 (Commit) 진행**
   - 로컬의 변경사항(파일 63개)을 스테이징(`git add .`)하고 `Initial commit of DNS Management Web App` 메시지와 함께 모두 안전하게 커밋 완료.
   - 기본 브랜치명을 `main`으로 설정(`git branch -M main`).

3. **원격 저장소 연동 및 푸시 (Push) 단계**
   - `git remote add origin https://github.com/youngminkim01/dns-web.git` 명령어를 통해 제공받은 GitHub 원격 주소 등록.
   - 현재 생성된 명령어 실행 대화창 권한을 사용자가 허용(승인)하는 즉시 로컬 소스코드가 모두 원격 저장소 `main` 브랜치로 업로드됩니다.

## 향후 관리 가이드
- 앞으로 코드를 변경하실 때마다 아래 명령어로 수시로 백업을 진행해 주세요.
  ```bash
  git add .
  git commit -m "작업 내용 설명"
  git push
  ```
