## 2026-02-10 DNS 관리 웹(대시보드) 문제해결 작업 정리

### 목표
- DNS 서버(BIND) 존 파일에 존재하는 레코드가 **DNS 관리 웹 화면(레코드 목록)**에 표시되지 않는 문제 해결

---

### 증상
- 웹 UI에서 **DNS 레코드 목록이 비어 있음**
- 서버는 구동되는 것 같으나 `/api/dns/records` 결과가 비거나 에러로 인해 빈 배열 반환

---

### 원인 및 해결 내용

#### 1) 프론트엔드 쿼리 파라미터 오타/불일치
- **원인**: 프론트가 `/api/dns/records?zones=...`로 호출했지만, 서버는 `req.query.zone`만 읽음
- **해결**: `dashboard.js`의 호출을 `zone`으로 수정
- **변경 파일**: `dashboard.js`

#### 2) SSH에서 sudo 비대화식 실행 실패
- **원인**: `sudo cat ...` 실행 시 TTY/비밀번호 입력 요구로 실패
- **해결**: `sudo -S -p ''`를 사용하고 `node-ssh`의 `execCommand(..., { stdin })`로 비밀번호를 전달하도록 수정
- **변경 파일**: `server.js`

#### 3) Windows에서 실행 시 리눅스 경로가 깨지는 문제(path.join)
- **원인**: 서버(Express)가 Windows에서 실행되며 `path.join()`이 원격(Linux) 경로를 `\`로 조립
  - SSH 명령에서 역슬래시가 이스케이프/탈락되며 경로가 `etcbindzonesdb.aproele.com` 형태로 깨져 “No such file” 발생
- **해결**: 원격 경로 조립은 항상 POSIX로 하도록 `remoteJoin()` 추가 후 `path.posix.join()` 사용
- **변경 파일**: `server.js`

#### 4) 존 파일 디렉토리 설정 정합성
- **확인된 실제 경로**: `/etc/bind/zones/db.aproele.com`
- **해결**: `server-config.js`의 `bind.zoneDir`를 `/etc/bind/zones`로 설정(존 파일 위치와 일치)
- **변경 파일**: `server-config.js`, `server.js`(존 디렉토리는 config를 사용하도록 반영)

#### 5) TTL 파싱(일부 레코드 TTL 생략 시)
- **원인**: 존 파일에서 TTL이 생략된 레코드(예: `dns01 IN A ...`)는 `parts[inIndex - 1]`가 TTL이 아니라 name이 될 수 있음
- **해결**: TTL 토큰이 숫자가 아니면 기본값 `3600`을 넣도록 보정
- **변경 파일**: `server.js`

---

### 검증(로컬에서 실제 실행/확인)
- Node 서버 실행: `npm start`
- API 호출 확인:
  - `GET http://localhost:3000/api/dns/records?zone=aproele.com`
- 결과:
  - SSH 연결 성공
  - 존 파일을 정상적으로 읽고 파싱하여 `records` 배열에 A 레코드가 반환됨
  - 예: `dns01`, `erp`, `ms` 등의 레코드가 JSON으로 응답됨

---

### 변경된 파일 목록
- `dashboard.js`
  - `/api/dns/records` 호출 시 query key를 `zone`으로 수정
- `server.js`
  - sudo 비대화식 실행(`stdin`) 처리
  - 원격 경로 조립을 POSIX로 고정(`path.posix.join`)
  - `bind.zoneDir` 설정 사용
  - TTL 파싱 보정(숫자 아니면 3600)
- `server-config.js`
  - `bind.zoneDir: '/etc/bind/zones'`로 설정

---

### 참고/추가 개선 아이디어(선택)
- **보안**: `server-config.js`에 SSH 비밀번호가 평문으로 존재 → `.env`로 분리 권장
- **파서 개선**: BIND 존 파일 포맷(상대/절대 도메인, `$TTL`, multiline 등) 지원 강화하면 안정성 향상
