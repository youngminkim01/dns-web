# DNS 관리 웹 페이지 관리자 매뉴얼

본 매뉴얼은 사내 DNS 자원을 손쉽게 관리할 수 있도록 구축된 **DNS 관리 웹 대시보드**의 사용 방법과 관리자 지침을 안내합니다.

---

## 1. 로그인 (접속하기)

1. 브라우저 주소창에 `http://dns.aproele.com` (또는 할당된 IP 주소 및 포트)을 입력하여 접속합니다.
2. 메인 화면에 나타나는 로그인 창에서 **관리자 비밀번호**를 입력합니다.
3. [로그인] 버튼을 클릭하거나 `Enter` 키를 눌러 대시보드 화면으로 이동합니다.

---

## 2. 대시보드 메인 화면 이해하기

로그인에 성공하면 **조회된 DNS 레코드 관리** 화면이 나타납니다.

* **좌측 메뉴바 (사이드바)** 
  * "조회된 DNS 레코드" 메뉴를 통해 현재 등록된 모든 레코드 목록을 볼 수 있습니다.
  * 하단 사용자 프로필 옆의 **[로그아웃 아이콘]**을 클릭하면 안전하게 로그아웃할 수 있습니다.
* **상단 검색창**
  * 수많은 레코드 중에서 원하는 레코드를 빠르게 찾을 수 있습니다.
  * 레코드의 **타입**, **이름**, 또는 **값(IP 등)**을 입력하면 즉시 검색 결과가 필터링됩니다.
* **레코드 목록 (테이블)**
  * **타입**: A, CNAME, TXT, MX 등 DNS 레코드 종류
  * **이름**: 설정된 도메인/서브도메인 이름 (예: `www.example.com`)
  * **값**: 연결된 IP 주소 또는 목적지
  * **상태**: 현재 정상 등록(Active) 상태 여부 
  * **작업**: 수정(✏️) 및 삭제(🗑️) 버튼

---

## 3. DNS 레코드 추가하기

새로운 서브도메인을 만들거나 새로운 서버 IP를 포인팅해야 할 때 사용합니다.

1. 화면 우측 상단의 **[새 레코드 추가]** 버튼을 클릭합니다.
2. "새 DNS 레코드 추가" 팝업(모달) 창이 나타납니다.
3. 다음 항목들을 빈칸 없이 입력합니다:
   * **레코드 타입**: 등록할 레코드의 타입을 설정합니다 (예: 서버 IP를 연결하는 기본값인 `A`).
   * **레코드 이름**: 연결하려는 도메인 또는 서브도메인을 입력합니다 (예: `api.example.com`).
   * **값**: 연결될 목적지의 IP 주소나 문자열을 입력합니다 (예: `192.168.10.100`).
   * (*MX 레코드의 경우 우선순위(Priority)를 추가로 설정할 수 있습니다.*)
4. 입력이 완료되면 **[저장]** 버튼을 클릭합니다.
5. "성공적으로 추가되었습니다." 메시지가 뜨면 DNS 서버망에 즉각적으로 반영됩니다.

---

## 4. DNS 레코드 수정 및 삭제

기존에 등록된 레코드의 IP가 변경되었거나 사용하지 않게 되어 삭제해야 할 때 활용합니다.

### 🔹 레코드 수정하기 (✏️)
1. 목록에서 변경을 원하는 레코드 우측의 **파란색 수정 버튼(✏️)**을 클릭합니다.
2. 팝업 창이 나타나면 변경하고자 하는 **값(IP 주소 등)**으로 내용을 수정합니다. (도메인 이름 등은 수정이 안될 시 삭제 후 재등록을 권장합니다.)
3. **[저장]** 버튼을 클릭하여 변경 사항을 서버에 반영합니다.

### 🔹 레코드 삭제하기 (🗑️)
1. 목록에서 더 이상 사용하지 않는 도메인 레코드의 우측 **빨간색 삭제 버튼(🗑️)**을 클릭합니다.
2. 정말 삭제할 것인지 묻는 경고 창이 나타납니다.
3. **[확인]**을 누르면 해당 DNS 레코드가 즉시 삭제됩니다.
   * *주의: 삭제된 레코드는 복구가 불가능하니 신중하게 삭제를 진행해 주세요.*

---

## 5. 자주 묻는 질문 (FAQ) & 주의사항

**Q: 추가/수정/삭제를 하면 언제부터 도메인 접속이 가능한가요?**
> A: 관리 웹에서 작업을 완료(저장)하면 **사내 DNS 서버(BIND)**에는 즉시 반영됩니다. 

**Q: 접속이 끊어지거나 "서버와 통신할 수 없습니다"라는 오류가 뜹니다.**
> A: 사내 네트워크망에 정상적으로 연결되어 있는지(VPN 연결 등) 확인하시기 바랍니다.

**Q: 레코드 타입을 변경하고 싶은데 선택이 되지 않습니다.**
> A: 현재 보안 및 시스템 안정성을 위해 주요 레코드(A) 외 다른 타입 선택이 제한되어 있을 수 있습니다. 타 레코드 등록이 필요할 경우 서버 시스템 관리자에게 문의해 주세요.
